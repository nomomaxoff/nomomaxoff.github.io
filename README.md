# Proyecto Web: Pokémon

**Temática:** Pokémon

**Usuarios:**

* Fan de Pokémon que busca información básica y visual.
* Usuario principiante que quiere entender conceptos básicos y tipos de Pokémon rápidamente.

**Requisitos:** GIFs "interactivos", navegación mediante anclas (`<nav>`) y estructura clara con `<header>`, `<footer>` y `<div>`.

**Distribución de tareas:** HTML, CSS, coordinación y animaciones.

**Diseño e interacción:** La página utiliza arquitectura cliente-servidor sencilla: el navegador solicita y muestra contenido HTML, CSS y GIFs desde GitHub Pages. El usuario navega mediante el menú del encabezado y visualiza contenido multimedia en secciones informativas y glosario.

**Despliegue:** Al principio, la carpeta de proyecto impedía la visualización del `index.html`, por lo que se reorganizaron los archivos en la raíz (`main`) para que GitHub Pages funcionara correctamente. Se utilizó GitHub Desktop para manejar pull, push y commits, explicando al equipo el flujo de trabajo. Una vez comprendido el funcionamiento de GitHub, el trabajo en equipo resultó dinámico, rápido e intuitivo.

**Mantenimiento y mejoras futuras:**

* Errores iniciales: dificultad para entender y realizar commits, push y pull; estructura de carpetas incorrecta; ajustes de márgenes y posiciones para evitar solapamientos.
* Mejoras futuras: añadir más animaciones y GIFs interactivos, contenido adicional sobre otros Pokémon, optimización para móviles y accesibilidad, e implementación de una barra de búsqueda.
